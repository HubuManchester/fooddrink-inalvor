# mockapi.io Data Source Configuration

The project supports reading and adding food data from mockapi.io.

## 1. Current Data Source

The project originally used locally mocked data in:

```text
FoodDrinkApp/Services/FoodCatalogService.cs
```

The current behaviour is:

- Prioritise the mockapi.io REST API
- If no API address is configured, or the network is temporarily unavailable, fall back to local sample data so the app never crashes

## 2. Create a Data Table on mockapi.io

1. Open [mockapi.io](https://mockapi.io)
2. Create a new Project or enter an existing one
3. Click Add Resource
4. Suggested Resource name:

```text
foods
```

5. Add the following fields:

| Field | Suggested Type | Description |
|---|---|---|
| name | String | Food or drink name |
| category | String | Category, e.g. Breakfast, Lunch, Drink |
| description | String | Description |
| calories | Number | Calories |
| protein | Number | Protein |
| carbs | Number | Carbs |
| fat | Number | Fat |
| allergyNote | String | Allergy note |
| tags | String | Search tags |

The `id` field is auto-generated by mockapi.io — do not add it manually.

## 4. Add the API Endpoint to the Project

After creating the Resource, mockapi.io generates an address similar to:

```text
https://682xxxx.mockapi.io/api/v1/foods
```

Open:

```text
FoodDrinkApp/Services/MockApiConfig.cs
```

Change `EndpointUrl` to your address:

```csharp
public const string EndpointUrl = "https://682xxxx.mockapi.io/api/v1/foods";
```

After re-running the app:

- The main list will fetch food data from mockapi.io
- The add-item page will POST to mockapi.io on save
- The detail page will fetch individual items via the API
